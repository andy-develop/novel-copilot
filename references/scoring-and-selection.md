## 7.1 方向自动评分选择

> 不再等人选方向，AI 按多维评分自动选择。

每个候选方向按以下维度评分（每项 1-5 分）：

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **节奏适配** (rhythm_fit) | 25% | 方向是否符合当前弧线阶段的需求？起势期选D3/D12，升压期选D7/D10，假高潮用D8，真高潮用D9/D13 |
| **翻页驱动** (drive_intensity) | 20% | 翻页驱动力强度（1=平淡，5=放不下）；焦虑型和情感型驱动加分 |
| **线索推进** (thread_advancement) | 15% | 是否推进了长期停滞的线索？是否收束了快到期的伏笔？推进冷线索额外+1 |
| **情感利用** (emotional_utilization) | 15% | 是否利用了未清偿的情感债？是否制造了新的情感债？回收+1，制造+0.5 |
| **一致性安全** (consistency_score) | 10% | 方向的逻辑风险等级：无风险5分，黄灯3分，红灯1分（红灯方向除非极高分否则不选） |
| **新颖度** (novelty) | 10% | 与近5章使用的发散模式是否重复？新模式5分，曾用过-1（D4反转60章内重复-2） |
| **钩子强度** (hook_strength) | 5% | 章末钩子的悬念/焦虑/认知冲击力度 |

**综合评分公式**：
```
Score = 0.25*rhythm_fit + 0.20*drive_intensity + 0.15*thread_advancement 
      + 0.15*emotional_utilization + 0.10*consistency_score + 0.10*novelty 
      + 0.05*hook_strength
```

**选择策略**：
- 最高分 ≥ 次高分 + 0.5 → 选最高分
- 最高分与次高分差距 < 0.5 → 合并两个方向的亮点，创建融合方向
- 最高分 < 2.5 → 暂停，在输出中标记"方向质量不足"，建议调整节奏

**排雷规则**（一票否决）：
- 红灯一致性风险 + severity=hard 规则冲突 → 排除
- 连续3章同一发散模式 → 排除
- 在假高潮位置选择了纯认知型模式（需要情感爆发）→ 排除

### 第1-2章特殊评分规则

> 读者给一本新小说的耐心只有两章。开篇的评分权重必须重聚焦：**翻页驱动和钩子强度才是生死线**。

当 `current_chapter ≤ 2` 时，评分权重 **大幅调整**：

| 维度 | 标准权重 | 第1-2章权重 | 理由 |
|------|---------|------------|------|
| **翻页驱动** (drive_intensity) | 20% | **35%** | 开篇首要目标：让读者翻页 |
| **钩子强度** (hook_strength) | 5% | **25%** | 章末钩子是开篇的生死线 |
| **节奏适配** (rhythm_fit) | 25% | 10% | 开篇不需要适配弧线节奏，需要制造节奏 |
| **情感利用** (emotional_utilization) | 15% | 15% | 不变，情感投资同样关键 |
| **线索推进** (thread_advancement) | 15% | 5% | 开篇线索尚未建立，推进权重降低 |
| **一致性安全** (consistency_score) | 10% | 5% | 开篇设定少，冲突风险低 |
| **新颖度** (novelty) | 10% | 5% | 新颖度重要但不如钩子生死攸关 |

**第1-2章评分公式**：
```
Score_ch1-2 = 0.10*rhythm_fit + 0.35*drive_intensity + 0.05*thread_advancement
            + 0.15*emotional_utilization + 0.05*consistency_score + 0.05*novelty
            + 0.25*hook_strength
```

**第1-2章额外一票否决**：
- 翻页驱动 ≤ 2 → 直接否决，无论总分多高
- 钩子强度 ≤ 2 → 直接否决，章末钩子是开篇的命脉
- 章节前500字无冲突/异常/紧迫感 → 直接否决

**第1-2章推荐发散模式**（优先级排序）：
1. **D7 认知差** — 读者立刻知道角色不知道的危险，30秒内制造焦虑
2. **D10 倒计时** — 开篇就倒计时，读者跟着紧迫
3. **D3 禁区** — 开场就在危险之地或正在做危险之事
4. **D12 暗流** — 看似正常的场景藏着3个微异常信号

## 7.2 一致性门控机制

> 不是每章都校验，而是在关键节点自动触发。

**触发条件**（满足任一即触发）：

| 触发类型 | 条件 | 校验深度 |
|----------|------|---------|
| 周期轻检 | 每 N 章（默认3章） | `check_all()` 快速校验 |
| 周期深审 | 每 M 章（默认10章） | `audit()` 深度审查 |
| 重大事件 | 角色死亡、地点毁灭、势力剧变 | `check_all()` + 专项 |
| 红灯告警 | 上一轮校验有 🔴 错误 | `check_all()` 确认修复 |
| 弧线切换 | 从一个弧线阶段进入下一个 | `audit()` + 节奏评估 |

**自动修复策略**：

| 错误类型 | 修复策略 |
|----------|---------|
| 死者行动（R1） | 回溯最近章节中该角色的行动，标注为回忆/幻觉/他人转述，或修改死亡状态为诈死（需有伏笔） |
| 物品双重持有（R3） | 以最新的 inventory_history 为准，自动修正另一方的 inventory |
| 地理矛盾（R4） | 插入过渡描写（旅途/传送），补充 timeline 时间标记 |
| 遗忘伏笔（W1） | 在下一章的 chapter-plan 中强制植入线索推进 |
| 角色缺席（W2） | 在下一章中安排该角色的提及或短暂出场 |

**修复后验证**：修复后重新运行 `check_all()` 如仍有错误，输出修复失败报告并继续（带修复标记）。

## 7.3 自动写作循环详解

**第一步：读取与计算**
```
1. 读取所有 state/*.yaml + novel-project.yaml + automation-config.yaml
2. 计算：
   - current_chapter：当前写到第几章
   - current_words：已写总字数
   - arc_phase：根据 chapter_plan 和全局弧线推算当前节奏阶段
   - stale_threads：超过 N 章未推进的线索
   - absent_chars：超过 M 章未出场的存活角色
   - recent_patterns：近 5 章使用的发散模式列表
   - emotional_debts_outstanding：未清偿的情感债
```

**第二步：发散**
```
1. 基于当前势能点（stale_threads、关系张力、未清偿情感债），从 14 种模式中选择 3-5 种最合适的
2. 每种模式生成一个候选方向，按发散输出格式填充
3. 特别注意：
   - 优先推进 stale_threads 中的 major 线索
   - 避免 recent_patterns 中出现过的模式
   - 在弧线转折点（假高潮、真高潮）必须使用高翻页驱动模式
```

**第三步：评分与选择**
```
1. 对每个候选方向按 7.1 的评分公式打分
2. 执行排雷规则
3. 选择最高分方向（或融合方向）
4. 生成 chapter-plan.yaml
```

**第四步：写作**
```
1. 根据 chapter-plan.yaml 生成章节正文
2. 写入 chapters/ch{NNN}.md
3. 确保每章结尾有钩子（微型钩子规则不变）
4. 字数控制在 avg_words_per_chapter ± 20% 范围
```

**第五步：状态更新**
```
1. 从新章节提取：
   - 角色行动、位置变化、新角色出场
   - 新线索 planted / 旧线索 developed / resolved
   - 死亡/受伤/修为变化
   - 物品转移
   - 关系变化
   - 时间线事件
2. 更新对应 yaml 文件
3. 更新 novel-project.yaml 的 current_chapter 和 current_words
4. 更新 arc_phase（如需要）
```

**第六步：一致性门控**
```
1. 判断是否触发校验（见 7.2 触发条件）
2. 如触发 → 运行校验 → 如有错误 → 自动修复 → 验证修复
3. 输出校验摘要到 state/consistency-log.yaml
```

**第七步：终止检查**
```
满足以下任一条件则终止循环：
- current_words >= target_words
- current_chapter >= target_chapters（如设定）
- 所有 major 线索已 resolved，且当前弧线处于"落幕"阶段
- 连续 3 次方向评分 < 2.5（创意枯竭信号）

未满足 → 回到第一步继续
```

## 7.4 循环中的节奏自调节

> 节奏不是预设的固定波形，而是根据叙事张力自适应调节。

**张力检测规则**：

| 信号 | 判断 | 调节 |
|------|------|------|
| 连续3章同一类型钩子 | 读者抗性形成 | 下一章必须换钩子类型 |
| 连续2章高张力(8+) | 读者疲劳 | 插入1章节奏缓冲（D6/D12轻度） |
| 连续3章低张力(<4) | 节奏拖沓 | 下一章必须选高翻页驱动模式(D7/D10) |
| major线索停滞>15章 | 叙事脱轨 | 下2章内必须推进 |
| 弧线已进入"落幕">3章 | 抓紧收束 | 确认所有major线索有归宿，否则延长落幕 |

**弧线自动推进**：每个弧线阶段有预计章节数配置（见 automation-config.yaml），超过预计章数的80%时开始向下一阶段过渡。

## 7.5 长线管理（跨弧线）

> 一部小说可能包含多个弧线，需要跨弧线管理全局线索和情感债。

**弧线间的衔接规则**：
- 上一弧线的落幕章必须种下下一弧线的悬念种子（至少1条新 major 线索）
- 跨弧线的 major 线索在每个弧线中至少推进1次
- 角色在弧线间的成长必须体现在 abilities/relationships/key_traits 的变化中
- 每个新弧线开始时，回顾上一弧线的 consistency-log，确认无遗留问题

**卷间过渡**：当一卷完结、新一卷开始时，不只是"继续写下一章"——这是叙事断代。详细操作步骤见 `references/auto-writing-execution-protocol.md` 的"卷间过渡协议"。核心操作：重置 arc-tracker → 封卷旧线索 → 播种新线索 → 演化角色 → 新增情感债 → 更新项目元信息 → 一致性确认。

**终局管理**（当接近目标字数的80%时）：
- 清点所有未收束的 major 线索
- 按重要性排序，为每条线索安排收束章
- 情感债回收优先级：命悬一线的承诺 > 长期思念 > 遗憾
- 避免最后5章内引入新的 major 线索
- 最终章必须有一个"概念回响"——呼应第1章的核心意象

## 7.9 方向融合技术

> 当评分脚本的最高分与次高分差距 < 0.5 时，融合两个方向的亮点通常优于单选最高分。

**融合策略**：
- 取最高分方向的发散模式作为主轴（节奏适配度最高）
- 从次高分方向中抽取1个互补模式（通常是情感或线索更强的那个）
- 融合后标注为 "D主+D副1+D副2" 格式

**实战验证的融合案例**：

| 融合 | 章节 | 效果 |
|------|------|------|
| D11+D5+D9 | 11 | 逆风孤战+规则突变+两难——B级权限被撤（D5），天盾选择是否暴露（D9），肉身挡执行者（D11）三线交织 |
| D12+D13+D7 | 12 | 暗流涌动+情感回收+认知差——苏晚晴验证中暗藏0x7F3B（D12），"我在搜索你"情感债回收（D13），天盾面临审计（D7） |
| D5+D9+D11 | 13 | 规则突变+两难+逆风——紧急维修漏洞突破（D5），天盾受限的两难（D9），孤身潜入核心层（D11） |

**注意**：融合3个以上模式容易导致章节焦点涣散。3个是上限。2个最佳。

**全自动模式的关键节奏参数**（验证有效的值）：
- 起势 4-5 章即可过渡升压，不宜拖到 8 章
- 升压期每章张力+1 到 +2，到第 6-7 章时张力可达 8-9
- 假高潮在第 8-10 章出现较理想——表面成功但暗藏危机
- 连续 2 章焦虑型钩子后插入 1 章悬念型或认知型，避免读者疲劳
- D7+D10 是最强焦虑组合，升压期可连续使用但不超过 2 章
