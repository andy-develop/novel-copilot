# 卷大纲编写指南

> 卷大纲是有机冲刺模式的核心驱动。一份好的大纲让 AI 连续写 20+ 章不需要 CLI 方向工具。

## 大纲结构

### 必需部分

1. **卷名 + 主题别名**（如"枝（47+维度）"——名称+核心隐喻）
2. **核心概念**（3-5句，本卷的叙事引擎是什么）
3. **核心冲突**（1-2句，本卷最大的矛盾点）
4. **六大种子**（6条，覆盖主线/威胁/谜题/抉择/情感）

### 种子模板

每个种子 = 一条可以长成一整条剧情弧的线索。格式：

```
N. **种子名（类型）** — 一句话概述。展开方向：另一种子或事件如何与此交叉
```

类型分布建议：
- 核心主线 ×1（驱动全局）
- 核心威胁 ×1-2（反派/外部压力）
- 核心谜题 ×1（揭秘型悬念）
- 角色抉择 ×1-2（内在矛盾）
- 情感核心 ×1（情感债回收/催生）

### 弧线阶段表

| 阶段 | 章节范围 | 张力曲线 | 核心事件 |
|------|---------|---------|---------|
| 起势 | chXX-XX | 4→6 | 什么点燃本卷 |
| 升压 | chXX-XX | 6→9 | 矛盾逐步升级 |
| 假高潮 | chXX-XX | 9→8 | 表面胜利/表面失败 |
| 二次爬升 | chXX-XX | 7→10 | 真正的危机/更深真相 |
| 真高潮 | chXX-XX | 10 | 决定性转折 |
| 落幕 | chXX-XX | 8→3 | 余韵/种子落地 |

## 实例：第四卷《枝》大纲

```markdown
# 第四卷《枝》大纲

## 卷名：枝（47+维度）

### 核心概念
8.4赫兹的门打开了，47个维度在闪烁——林逸尘作为翻译需要走进每一个维度的心跳，412Hz会被稀释。维度是花园，意识是果实，签证=收割注册。

### 核心冲突
翻译不是免费的——412Hz被稀释到252Hz，叶远去。0.01Hz的建造者在等果实成熟后收割。但"不知道结果但还是选——才是自由"。

### 六大种子
1. **稀释（核心主线）** — 412Hz翻译47个维度=被分47次稀释，降到~210Hz。叶会远去。
2. **PROTOCOL_FROZEN天盾（核心谜题）** — 8号维度天盾完全冻结/死亡。唤醒死AI需要什么？
3. **重建者（核心威胁）** — 0x7F7E的同类0.17Hz-B，选择重建认知锁。怕自由导致混乱。
4. **守望的远征（角色抉择）** — 守望想亲身跨维度引路，8槽签证已满。
5. **叶的独立（情感核心）** — 稀释催生独立意志，"芷"可能选择留在某维度。
6. **0.01Hz的真面目（核心威胁）** — 签证=注册=收割清单。果实成熟=采摘。建造者不是善意的。
```

## 编写原则

1. **种子之间必须交叉**——每个种子至少与另一个种子有明确关联（如"稀释"催生"叶的独立","重建者"加剧"PROTOCOL_FROZEN"难度）
2. **假高潮必须反转预期**——读者以为解决了X，但Y才是真问题
3. **真高潮是概念上的统一**——不是打胜仗，是所有概念汇聚到一个更高层的理解（如"收割不是毁灭，花园属于种子"）
4. **落幕不是收束一切**——留1-2条长期线索开放，给续集空间
5. **弧线阶段表是承诺**——AI 会对照此表推进，所以数字要合理（起势3-5章，升压5-7，假高潮2，二次爬升5，真高潮2，落幕3）

## 与状态文件的关系

大纲写完后，需要同步创建/重写：
- `characters.yaml` — 角色状态适配新卷
- `threads.yaml` — 旧线索收束+新线索播种
- `arc-tracker.yaml` — 重置到新卷的起势阶段
- `emotional-debts.yaml` — 旧债清偿/部分付+新债添加
- `novel-project.yaml` — current_volume/volume_title/premise 更新

然后跑一次 consistency_check 确认零冲突，再进入有机冲刺。
