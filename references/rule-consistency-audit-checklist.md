# 规则一致性审计清单

> 当修改 novel-copilot 的任何规则时，使用本清单检查跨文件一致性。
> 本 skill 的规则分布在 4+ 个文件中（SKILL.md + references/），同一概念在不同文件中出现时极易产生阈值、措辞、推荐方向的分歧。

## 何时使用

- 修改了评分权重、一票否决条件、章节范围约束
- 修改了钩子类型推荐、发散模式推荐、弧线阶段对应
- 新增了适用于特定章节范围的规则
- 任何涉及数字阈值（字数、章数、百分比）的改动

## 审计维度

### 1. 数值阈值漂移

同一概念在多处出现时，数值必须一致。

| 检查项 | 文件位置 | 正确值 |
|--------|---------|--------|
| 纯设定段落上限 | §8.4 审查清单 + §9.1 法则二 + Step 4.5 + Step 4.6 | **200字** |
| 入局字数 | §9.1 法则一 + §8.3 一票否决 + Step 4.6 | **300字**（开篇前300字入局） |
| 高潮分拆阈值 | §9.2 纪律二 + Step 4 + 排雷规则 | **5000字** |
| 连续被动干预 | §9.3 红线 + 能动性原则 + 排雷规则 | **2章→干预，3章→否决** |
| 钩子连续禁用 | divergence 层级三 + §7.4 张力检测 | **2章** |

### 2. 严重性级别混淆

规则应有清晰的**两级体系**，不可混为一谈：

- **⚠️ 干预级**：发生了就需要纠正，但不阻止继续（如连续2章被动→下章必须主动）
- **⏹️ 否决级**：方向直接排除/规则绝对禁止（如连续3章被动→一票否决，主角被剥夺核心能力→红线）

检查：每个"禁止"规则的严重性在所有文件中是否一致（❌=红线禁止 vs ⚠️=干预纠正）。

### 3. 多维度推荐冲突

当规则从不同维度给出推荐时（如"章节范围推荐" vs "弧线阶段推荐"），必须声明优先级。

- **当前约定**：弧线阶段推荐优先，章节范围推荐作为加分项
- 检查点：如果第3-10章推荐D11但弧线在起势阶段（D11不适合起势），应以弧线阶段为准
- 修改任一推荐列表时，必须检查另一维度是否冲突

### 4. 钩子类型推荐一致性

每个弧线阶段的钩子类型推荐在三处出现，必须一致：

| 弧线阶段 | 正确推荐 | divergence-patterns 弧线表 | execution-protocol 弧线表 | 钩子使用原则 |
|---------|---------|--------------------------|--------------------------|------------|
| 起势 | 认知型 | ✅ | ✅ | — |
| 升压 | 悬念型 | ✅ | ✅ | — |
| 假高潮 | 认知型为主+情感型 | ✅ | ✅ | ✅ (原则3) |
| 二次爬升 | 悬念+情感型 | ✅ | ✅ | — |
| 真高潮 | 全类型爆发 | ✅ | ✅ | — |
| 落幕 | 好奇型为主+情感型 | ✅ | ✅ | ✅ (原则4) |

### 5. 发散模式力度级别区分

D4（反转颠覆）有三个力度级别，禁忌/推荐必须标注级别：

- **局部反转**：中冲击，可用多次，任何弧线阶段可用
- **结构反转**：高冲击，每弧线1-2次，假高潮/真高潮可用
- **存在论反转**：最高冲击，全书1-2次，**前10章禁忌**（世界观基础未建立）

检查：任何"推荐D4"或"禁忌D4"的条目是否标注了力度级别。

### 6. 章节范围规则的边界情况

当规则按章节范围分段（第1-2章 / 第3-10章 / 第11章+）时，检查：

- **边界值**是否明确：`≤2` 还是 `<3`？`3≤x≤10` 还是 `3-10章`？
- **过渡**是否平滑：第3章从ch1-2权重切换到ch3-10权重，是否跳变太大？
- **重叠**是否冲突：第1-2章的冲击力审查和第3章起的叙事自检，对同一检查项（如设定段落上限）是否一致？

## 快速审计流程

```
1. 列出本次修改涉及的规则关键词
2. 在全部4个文件中搜索该关键词
3. 对比每处出现的数值、阈值、严重性级别
4. 如发现不一致 → 以最严格的值统一（除非有明确理由放宽）
5. 检查钩子类型推荐表和弧线阶段推荐表是否同步
6. 修改后 git push
```
