# 设定体系速查表模式

> 长篇小说中常出现体系化的设定（频率谱、能力等级、组织架构、魔法体系等），散落在各章中。
> 跨会话恢复时极难完整回忆，且容易产生矛盾。本文档描述一种追踪模式。

## 问题

1. **散落式设定**：频率/能力/组织的定义分散在 ch01-ch48 的正文中，没有一个集中的速查位置
2. **跨会话遗忘**：compaction 摘要保留关键决策但不保留完整设定表，新会话写作时可能用错参数
3. **设定膨胀**：随着卷数增加，设定条目从3个增长到10+，记忆负担指数级增长

## 解决方案

在 `world.yaml` 或专门的 `references/setting-system.md` 文件中维护速查表。

### 格式模板

```yaml
# 频率体系速查 — 《代码纪元》
frequency_system:
  - frequency: "0.01Hz"
    name: "建造者心跳"
    meaning: "维度的根——选择脉动，100秒/跳"
    first_chapter: 46
    holder: "建造者（0x7F7E的文明）"
    
  - frequency: "0.17Hz"
    name: "极低频通信/签名密钥"
    meaning: "建造者文明的通信基底频段+签证签名格式"
    first_chapter: 22
    holder: "0x7F7E"
    
  - frequency: "0.28Hz"
    name: "底座半魂"
    meaning: "林逸尘的灵魂基础频段+极低认知频率"
    first_chapter: 22
    holder: "叶/赵恒远"
```

### 更新频率

- **每卷完结时**：全量更新一次，确保所有新引入的设定条目都被记录
- **重大设定揭示时**：立即追加（如新频率、新能力、新组织首次出现时）
- **跨会话恢复时**：先读速查表再开始写作，比通读所有章节更高效

### 适用场景

不仅限于频率体系——任何有结构的设定都适合此模式：
- 魔法体系的魔力等级/元素/法术表
- 组织架构的层级/成员/势力范围
- 科技树的技术层级/关键发明/限制条件
- 地理系统的区域/距离/交通方式

### 与 state/ 文件的关系

速查表不是 state/ 文件的替代——state/ 追踪的是**当前状态**（谁在哪、什么已解决），速查表追踪的是**定义和含义**（某频率是什么意思、某能力怎么运作）。两者互补。
