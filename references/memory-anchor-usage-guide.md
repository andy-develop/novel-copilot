# 记忆锚点污染使用指南

## 为什么要用这个

AI 发散引擎以 threads.yaml 中的已知线索为种子。这导致一个问题：**创意被限制在已标记的框架内**。被遗忘的前文细节（threads.yaml 之外的东西）才是最大的创意矿藏。

## 六种可扫描的遗忘细节

| 类型 | 特征 | 典型出处 | 示例 |
|------|------|---------|------|
| 🗣️ **配角闲话** | 随口说的、与当前剧情无关、无后续 | 日常对话、路人台词 | "这柳树下的石碑我总觉得眼熟" @ch008 |
| 👀 **无动机行为** | 做了但没解释 | 眼神、动作、缺席 | "林姑娘看了主角一眼，转身走了" @ch012 |
| 🏺 **陌生物品** | 有描写但从未使用 | 场景装饰、配角携带物 | "桌上摆着一盏青铜灯，上面刻着奇怪的符文" @ch003 |
| 🚶 **无因果离场** | 提前离场/消失未提及 | 配角中途离开、无故缺席 | "王管事说有事要办，提前离席" @ch015 |
| 🌫️ **环境异常** | 非自然的天气/光影/声音 | 怪异描写但无后续 | "天色暗了一瞬，像是有什么东西遮住了太阳" @ch006 |
| 🤫 **未完成的话** | 被中断且未续上 | 欲言又止、被打断 | "其实那件事……算了，以后再说" @ch010 |

## 操作流程

```
1. 确定当前章 ch{N}
2. 扫描范围: ch001 ~ ch(N-2)（跳过上一章，太新了）
3. 对每个符合条件的细节标注: [类型] + [原文] + [来源]
4. 随机抽取 2-4 个
5. 对每个抽取的细节问: "如果这件事不是无意义的，它的意义是什么？"
6. 注入发散种子池，标注 seed_anchor
```

## 规则

- **同一细节 20 章内不重复抽取**
- **锚点可独立驱动方向，也可与其他模式组合**：
  - 组合时锚点提供「种子」，D1-D15 提供「结构」
  - 例：遗忘细节"林姑娘看石碑" + D3(禁区探索) → "林姑娘独自去了某个禁地"
  - 例：遗忘细节"王管事提前离场" + D15(反常配角) → "王管事离场后做了什么"
- **自然融合 → novelty 额外+1**：如果锚点种子与章节内容有机融合，不是强行塞入

## 自检

```
□ 挑选的细节是 "threads.yaml 未追踪" 的吗？
□ 这真的是细节，不是主线线索的遗漏标注？
□ 如果这个细节确实无意义——现在的发散方向还值得写吗？
   （有时候一个无意义细节就是无意义，不必强求每个都被回收）
```
