## ✨ 特性

### 🧠 智能剧情发散引擎（14 种模式）

不再"想到哪写到哪"。每章从 14 种发散模式中自动生成 3-5 个方向，7 维加权评分选出最优：

| 模式 | 核心 | 适用场景 |
|------|------|---------|
| D1 线索咬合 | 伏笔回收，多条线交汇 | 中期推进 |
| D2 关系裂变 | 角色关系突转 | 情感爆发 |
| D3 禁区探索 | 踏入危险未知 | 制造紧张 |
| D4 反转颠覆 | 颠覆已有认知 | 高潮爆发 |
| D7 认知差悬念 | 读者比角色知道更多 | 开篇/维持焦虑 |
| D10 倒计时压迫 | 时间紧迫，步步紧逼 | 制造紧迫 |
| D13 情感债回收 | 情感伏笔兑现 | 情感爆发 |

> 7 维加权评分：节奏适配 25% + 翻页驱动 20% + 线索推进 15% + 情感利用 15% + 一致性 10% + 新颖度 10% + 钩子 5%，确保选出的方向兼具爽感和叙事合理性。

---

### 🔒 逻辑一致性守护

> **写长篇最怕"写到后面忘了前面"。** 本系统通过结构化状态追踪保证全书一致性。

| 状态库 | 功能 | 防崩盘机制 |
|--------|------|-----------|
| `characters.yaml` | 角色状态（生死/位置/能力/关系） | 死人不可再行动，物品唯一持有 |
| `world.yaml` | 世界设定（地理/势力/能力体系） | 已毁地点不可引用 |
| `threads.yaml` | 剧情线索（伏笔/收束状态） | major 线索遗忘告警，已收束不重用 |
| `timeline.yaml` | 时间线（章节事件） | 角色不瞬移，时间流逝合理 |
| `rules.yaml` | 世界规则（硬规则不可违反） | hard 规则违反一票否决 |

每章写完自动校验 → 红灯自动修复 → Git 同步。**70+ 种逻辑问题自动检测。**

---

### 🔥 爽度引擎（六大爽型 × 三步构建）

> 网文读者买单的不是文笔，是"爽"。没有回报的付出是受罪，不是故事。

**六大爽型**：装逼打脸 | 逆风翻盘 | 智商碾压 | 实力碾压 | 夺宝抢机缘 | 人心归附

**爽点三步法**：压制/困境（让读者憋气）→ 转折/蓄力（合理翻盘）→ 释放/碾压（快准狠，2 段内完成）

**密度规则**：每 3 章 ≥1 中爽点，每 5 章 ≥1 大爽点，连续 2 章零爽点 → 触发节奏违规告警。

**旁观者三层反应**：爽点的 50% 力量来自围观者的震惊——直接对手崩溃 + 同级震惊 + 高手动容。

---

### 🎬 画面感引擎（六大技法）

> 画面感 = 读者闭眼能看见。没有画面的文字是报告，不是小说。

| 技法 | 效果 | 示例 |
|------|------|------|
| **镜头语言** | 远→中→特切换，制造电影感 | "城墙上只剩一个人影"（远景）→ "指甲下面全是干涸的血"（特写） |
| **感官五重** | 重要场景 ≥3 种感官，高潮 5 种 | 视觉+听觉+触觉+嗅觉+味觉 |
| **动作微分解** | 关键动作拆成 1-3 秒慢动作 | "剑从鞘中滑出——刀光一线——喉咙处绽开一道红线" |
| **环境投射** | 用天气/光线/空间说情绪 | 压迫→低矮天花板，觉醒→光线涌入 |
| **物件象征** | 一件物品走完三阶段，封印叙事使命 | 铜钱：捡起 → 硌着胸口 → 滚落坟前不捡 |
| **声画对位** | 声音和画面矛盾时张力最大 | 身后人声鼎沸，他只听得见心跳 |

**铁律：爽点必须有画面，画面必须服务爽点。**

---

### ⚡ 快节奏四原则

| 原则 | 规则 |
|------|------|
| **零废段** | 每段必须完成至少一项：推进剧情 / 制造冲突 / 揭示信息 / 创造爽点。否则删。 |
| **场景快切** | 场景切换不超过 3 句过渡 |
| **对话即角力** | 每句对话必须推进信息 / 施压 / 反击 / 暴露弱点 |
| **行动代心理** | 用行动和对话展现内心，心理独白超 150 字且无冲突穿插 → 违规 |

---

### 🛡️ 叙事纪律

- **300 字入局**：每章前 300 字必须进入冲突/行动，禁止环境描写/背景回顾开篇
- **高潮分拆**：大高潮超 5000 字必须拆为 2-3 章，最高潮处断章
- **前 10 章不虐主**：读者认同感决定追读率，禁止严重羞辱/剥夺核心能力/连续被动
- **开篇冲击力**：第 1-2 章专用评分规则，一票否决翻页驱动 ≤2 或钩子强度 ≤2

## 📦 安装

### 前置条件

- **[Hermes Agent](https://hermes-agent.nousresearch.com)** — 本 skill 是 Hermes Agent 的技能插件，需在 Hermes 环境中运行
- **Python 3.10+**（CLI 工具依赖）
- **GitHub CLI (`gh`)**（自动仓库创建和 Git 同步）

### 安装 Skill

```bash
# 从 GitHub 安装
hermes skills install https://github.com/andy-develop/novel-copilot --force

# 或克隆到本地 skills 目录
git clone https://github.com/andy-develop/novel-copilot.git ~/.hermes/skills/creative/novel-copilot
```

安装后，在 Hermes 对话中说"写小说"即可自动触发该 skill。

## 🚀 使用

### 全自动写小说（推荐）

只需三步，AI 自动完成剩余所有工作：

```bash
# 第一步：在 Hermes 对话中初始化项目
python3 scripts/auto_write.py init --name "我的小说名" --mode auto
```

```bash
# 第二步：创建 GitHub 仓库（版本控制 + 多会话同步）
cd ~/novels/我的小说名
git init && git config user.email "you@example.com" && git config user.name "Your Name"
git add -A && git commit -m "init: 项目初始化"
git branch -m main
gh repo create andy-develop/我的小说名 --private --source=. --push
```

```bash
# 第三步：对 Hermes 说出你的开头
# 在对话中说：
"全自动写《我的小说名》，开头如下：
[粘贴你的开头内容]"
```

之后 AI 自动进入循环：

```
发散 3-5 方向 → 7 维评分选最优 → 写章节正文 → 一致性校验 → Git 同步 → 继续下一章
```

**每章输出**：📝完成 / 🎭模式 / 🎣钩子 / 📊进度 / ✅一致性

---

### 中途干预

随时可以在 Hermes 对话中发出指令：

| 指令 | 效果 |
|------|------|
| "暂停" / "停" | 停止自动写作，进入人工模式 |
| "继续" / "下一章" | 恢复自动写作（连续冲刺，不问问题） |
| "这一章我来选方向" | 暂停自动选择，列出方向让你挑 |
| "用 D9 两难抉择" | 指定下一章使用特定发散模式 |
| "节奏太慢，加快" | 调整节奏参数，优先紧张模式 |
| "角色 X 不该死/应该活" | 修正角色状态 |
| "跳到高潮" | 加速推进到弧线高潮 |
| "卷完结" | 触发卷完结流程（全局审计 + 字数统计 + 线索收束） |

---

### 人机协作模式

如果你希望更多地参与创作决策：

```bash
# 初始化（不带 --mode 或指定 manual）
python3 scripts/novel_state.py init --name "书名"
# 然后对 Hermes 说"写第一章，方向：[你的想法]"
```

---

### 卷间过渡

长篇写完一卷后，自动处理：
1. `volume_transition.py` 机械重置状态
2. AI 自动创建新卷大纲（六种子 + 弧线阶段 + 逐章规划）
3. 全量重写角色库 / 线索库 / 情感债
4. 一致性确认 → Git 提交 → 开始新卷

| 命令 | 用途 | 示例 |

## 📊 技术架构

```
my-novel/                        # ← GitHub 仓库
├── state/
│   ├── characters.yaml          # 角色状态库
│   ├── world.yaml               # 世界设定库
│   ├── threads.yaml             # 剧情线索库（伏笔→收束）
│   ├── timeline.yaml            # 时间线
│   ├── rules.yaml               # 世界硬规则
│   ├── arc-tracker.yaml         # 弧线追踪
│   ├── emotional-debts.yaml     # 情感债追踪（情感伏笔）
│   └── consistency-log.yaml     # 一致性校验日志
├── chapters/                    # 章节正文 ch{NNN}.md
├── outlines/                    # 弧线大纲 + 章节规划
├── automation-config.yaml       # 自动写作配置（节奏/评分/冲刺）
└── novel-project.yaml           # 项目元信息
```

**核心循环**：

```
开头/种子 → 初始化项目 → 提取设定 → 填充状态库
  ↓
LOOP:
  读取状态 → 发散 3-5 方向 → 7 维评分 → 选择最优
  → 写章节正文 → commit_chapter 更新状态
  → 一致性门控（轻检/深审，红灯自动修复）
  → Git 同步 → 终止检查（否 → 继续）
```

## 🎯 叙事优先级

冲突时高优先级不可妥协：

| 优先级 | 内容 | 说明 |
|--------|------|------|
| **P0** | 节奏 + 爽度 + 画面感 | 读者留不留得住、爽不爽、记不记得住 |
| **P1** | 一致性 + 散文质量 | 不崩盘、不啰嗦 |
| **P2** | 大纲 + 字数 | 工具而非枷锁 |

> **当 P0 与 P2 冲突时，节奏爽感优先，调整大纲而非牺牲节奏。**
