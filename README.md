## 这是什么？

一套 **prompt + CLI 工具**，让 AI agent 替你写长篇网文。

你提供一个开头，AI 自动：发散剧情方向 → 评分选最优 → 写完一章 → 校验一致性 → 更新状态库 → 循环写下一章，直到全书完结。

**不绑定任何特定 agent 平台**——只要你的 AI 能读 markdown 文件、能执行 shell 命令，就能用。已实测兼容：

| Agent 平台 | 使用方式 |
|-----------|---------|
| **Hermes Agent** | 安装为 skill，对话中自然触发 |
| **Claude Code** | 加载 `SKILL.md` 为项目指令，`/init` 后说"继续"即可 |
| **OpenCode** | 将 `SKILL.md` 放入项目，agent 自动读取并执行 |
| **Codex CLI** | `codex exec` 配合 SKILL.md 指令驱动 |
| **OpenClaw** | 作为 project skill 加载 |
| **任意支持 terminal 的 AI** | 直接告诉它"按 SKILL.md 的流程写小说" |

> 本质：`SKILL.md` 是「剧本」（告诉 AI 怎么做），`scripts/` 是「道具」（CLI 工具），`templates/` 是「舞台」（状态文件模板）。三者搭配，任何 agent 都能当你的专属网文写手。

## ✨ 核心特性

### 🧠 14 种剧情发散模式——不再「想到哪写到哪」

| 模式 | 核心 | 适用场景 |
|------|------|---------|
| D1 线索咬合 | 伏笔回收，多条线交汇 | 中期推进 |
| D2 关系裂变 | 角色关系突转 | 情感爆发 |
| D3 禁区探索 | 踏入危险未知 | 制造紧张 |
| D4 反转颠覆 | 颠覆已有认知 | 高潮爆发 |
| D7 认知差悬念 | 读者比角色知道更多 | 开篇/维持焦虑 |
| D10 倒计时压迫 | 时间紧迫，步步紧逼 | 制造紧迫 |
| D13 情感债回收 | 情感伏笔兑现 | 情感爆发 |

每章从 14 种模式中生成 3-5 个方向，**7 维加权评分**自动选出最优（节奏 25% + 翻页驱动 20% + 线索推进 15% + 情感利用 15% + 一致性 10% + 新颖度 10% + 钩子 5%）。

---

### 🔒 逻辑一致性守护——死人不会复活

五套结构化状态库追踪全书，70+ 种逻辑问题自动检测：

| 状态库 | 职责 | 防崩盘规则 |
|--------|------|-----------|
| `characters.yaml` | 角色状态（生死/位置/能力/关系） | 死者不可行动，物品唯一持有 |
| `world.yaml` | 世界设定（地理/势力/能力体系） | 已毁地点不可引用 |
| `threads.yaml` | 剧情线索（伏笔→收束） | major 线索遗忘告警 |
| `timeline.yaml` | 事件时间线 | 角色不瞬移，时间合理 |
| `rules.yaml` | 世界硬规则 | 违反一票否决 |

每章写完自动运行 `consistency_checker.py`，红灯自动修复。

### 🔥 爽度引擎——读者买单的是「爽」

| 六大爽型 | 三步构建法 | 密度规则 |
|---------|-----------|---------|
| 装逼打脸 | ① 压制/困境 → 憋气 | 每 3 章 ≥1 中爽点 |
| 逆风翻盘 | ② 转折/蓄力 → 合理翻盘 | 每 5 章 ≥1 大爽点 |
| 智商碾压 | ③ 释放/碾压 → 2 段内完成 | 连续零爽 → 违规告警 |
| 实力碾压 + 夺宝机缘 + 人心归附 | 爽点 50% 力量来自旁观者震惊反应 |

---

### 🎬 画面感引擎——读者闭眼能看见

| 六大技法 | 一句话 |
|---------|--------|
| 镜头语言 | 远→中→特切换，「剑从鞘中滑出——刀光一线——喉咙绽开一道红线」 |
| 感官五重 | 重要场景 ≥3 种感官，高潮场景 5 种全满 |
| 动作微分解 | 关键动作拆成 1-3 秒慢动作子弹时间 |
| 环境投射 | 用天气/光线/空间说情绪，不用形容词 |
| 物件象征 | 一件物品走完三阶段，封印叙事使命 |
| 声画对位 | 声音和画面矛盾时张力最大 |

---

### ⚡ 快节奏四原则

- **零废段**：每段至少完成推进剧情/制造冲突/揭示信息/创造爽点之一，否则删
- **场景快切**：过渡不超过 3 句
- **对话即角力**：每句对话推进信息/施压/反击/暴露弱点
- **行动代心理**：用行动和对话展现内心，禁止大段独白

### 🛡️ 叙事纪律

- **300 字入局**：每章前 300 字必须进入冲突，禁止环境描写/背景回顾开篇
- **高潮分拆**：大高潮超 5000 字必须拆为 2-3 章，在最紧要处断章
- **前 10 章不虐主**：主角连续被动不超过 2 章，保护读者认同感
- **第 1-2 章冲击力引擎**：专用评分 + 一票否决，翻页驱动不达标直接重写

## 🚀 快速开始

### 三步全自动写小说

**第一步**：告诉你的 AI agent 加载 `SKILL.md` 作为工作指令。

**第二步**：让 agent 初始化项目：

```bash
python3 scripts/auto_write.py init --name "我的小说名" --mode auto
```

**第三步**：给 agent 一个开头，说「全自动写完」。

之后 agent 自动循环：

```
读取状态 → 发散3-5方向 → 评分选最优
→ 写章节正文 → 一致性校验
→ 更新状态库 → Git同步 → 继续下一章
```

每章输出：📝完成 / 🎭模式 / 🎣钩子 / 📊进度 / ✅一致性

## 🛠️ CLI 工具

所有工具都是独立 Python 脚本，任何 agent 直接调用即可：

| 工具 | 功能 | 示例 |
|------|------|------|
| `auto_write.py init` | 初始化项目，生成状态文件 | `--name "书名" --mode auto` |
| `auto_write.py wordcount` | 精确中文字数统计 | `--dir . --update` |
| `commit_chapter.py` | 章后一键更新所有状态 | `--chapter 42 --words 3500 --pattern "D5+D9"` |
| `consistency_checker.py check` | 章级一致性轻检 | `check <目录>` |
| `consistency_checker.py audit` | 卷级全局深度审查 | `audit <目录>` |
| `volume_transition.py` | 卷间机械状态重置 | `--volume 2 --title "卷名"` |
| `auto_sync.py` | 检测状态文件同步差距 | `--dir .` |
| `novel_state.py` | 项目状态管理 | `init / status / export` |

## 📂 项目结构

```

├── state/

│   ├── timeline.yaml            # 事件时间线
│   ├── rules.yaml               # 世界硬规则（不可违反）
│   ├── arc-tracker.yaml         # 弧线阶段追踪
│   ├── emotional-debts.yaml     # 情感债追踪
│   └── consistency-log.yaml     # 一致性校验日志
├── chapters/                    # 章节正文 ch{NNN}.md
├── outlines/                    # 弧线大纲 + 章节规划
├── automation-config.yaml       # 自动写作配置
└── novel-project.yaml           # 项目元信息
```

## 🎮 中途控制

自动写作过程中，随时对 agent 说出指令即可干预：

| 指令 | 效果 |
|------|------|
| 「暂停」 | 停止自动写作 |
| 「继续」 | 恢复自动写作（不问问题，连续冲刺） |
| 「用 D9」 | 下一章指定发散模式 |
| 「节奏太慢」 | 调整节奏参数 |
| 「角色 X 不该死」 | 修正角色状态 |
| 「跳到高潮」 | 加速推进到弧线高潮 |

## 🎯 叙事优先级

| 优先级 | 内容 | 原则 |
|--------|------|------|
| **P0** | 节奏 + 爽度 + 画面感 | 读者留不留得住、爽不爽、记不记得住 |
| **P1** | 一致性 + 散文质量 | 不崩盘、不啰嗦 |
| **P2** | 大纲 + 字数 | 工具而非枷锁 |

> P0 与 P2 冲突时，**节奏爽感优先**，调整大纲而非牺牲节奏。
